# 安全策略

## 支持范围

当前默认维护分支：`main`。

如果你发现安全问题，请不要直接公开提 Issue 或提交公开 PR 细节。

## 漏洞报告方式

请通过以下方式私下联系维护者：

- GitHub: https://github.com/Yichen-Gao
- Email: 2311438294@qq.com

建议邮件标题：`[Labflow Security] 简短问题描述`

## 报告内容建议

为了更快复现和修复，请尽量提供：

- 受影响版本/提交号
- 影响范围（可读/可写/提权/信息泄露等）
- 复现步骤
- 预期行为与实际行为
- 如有 PoC，请提供最小复现

## 响应与修复流程

- 72 小时内确认收到报告
- 先评估影响与复现
- 修复后发布补丁并在仓库说明
- 必要时补充迁移/缓解建议

## 敏感信息处理

请勿在公开渠道贴出：

- SMTP 授权码
- 真实服务器 IP / 内网拓扑
- 生产环境数据库或日志原文

如果误提交敏感信息，请立即撤销并轮换密钥。
